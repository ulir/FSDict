Detailed Description

template<typename MinDicType = MinDic< int >>
class fsdict::Vaam< MinDicType >

Vaam is a module for approximate matching in large dictionaries, considering spelling variants (defined with a set of rewrite patterns) and also spelling errors (using levenshtein distance).

Besides this class reference, please read the Vaam Manual to get an idea of how to create a Vaam object, how to send queries and how to interpret Vaam's answer.

As input the tool needs

  • a wordList baseDic compiled as FSA (use the tool compileMD to get such an automaton from a word list.
  • a file containing a list of patterns, one per line. A line containing 't th' indicates that a 't' in baseDic could be transformed to 'th'.
  • a threshold maxNrOfPatterns specifying the maximum number of pattern applications in one word (default: 1000, which is near to infinite ).
  • a distance threshold distance specifying the maximum number of edit operations allowed on top of the application of variant patterns (default: 0, allow variant patterns only).

Definition of the output:

We define a so-called hypothetical dictionary hypotheticalDic that contains all spelling variants which can be obtained by applying a sequence of up to maxNrOfPatterns variant patterns to any word of baseDic. One entry of hypotheticalDic is a triple {<word,baseWord,instruction>} where instruction is the above mentioned sequence to get the variant word from the original word baseWord from baseDic.

For a query q the algorithm returns all such triples {<word,baseWord,instruction>} from hypotheticalDic where the standard levenshtein distance between word and q does not exceed
