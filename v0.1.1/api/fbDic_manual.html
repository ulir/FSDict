FBDic Manual

On this page the use of "fbdic"s from the csl library is explained.

"fb" means forward-backward: This type of dictionary is used for approximate search with fsdict::MSMatch, which also needs the dictionary of all words in reverse order. FBDic is a simple wrapper that holds two data structures of type fsdict::MinDic, holding the dictionary in regular and in reverse order.

Use the program compileFBDic to create binary automata files from text lists

This section explains how to obtain an FBDic binary file from a text list.

In the directory INSTALL_PREFIX/bin you find a program compileFBDic. Use it like:

compileFBDic foo.lex foo.fbdic

A .lex file is a text file of the form:

anna#42
bill#50
charly#54

That is, an alphabetically sorted list of keys, followed by a '#' delimiter symbol and some integer value which is stored with the key. If delimiter and integer are omitted, the value 0 will be annotated with each key.

It is important that the .lex file is utf8 encoded and sorted properly. You can sort your dictionary with the tool sortLex included in this package. Much faster is the following unix command:

LC_ALL=C sort -t'#' -k1,1 foo.unsorted > foo.lex

The resulting .fbdic file is a binary representation of the dictionary automaton and can now be used with other csl tools.

Load a binary file into a fsdict::FBDic object

Loading a binary automaton file into a c++ object is easy:

fsdict::FBDic<> fbDic( "path-to/some.fbdic" );

or

fsdict::FBDic<> fbDic;
fbDic.loadFromFile( "path-to/some.fbdic" );

Some modules (like fsdict::DictSearch) also allow to pass a filename and load the automaton internally.

compile a fsdict::FBDic automaton from a .lex file at runtime

fsdict::FBDic<> fbDic;
fbDic.compileDic( "path-to/some.lex" );
